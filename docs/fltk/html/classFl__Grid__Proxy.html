An implementation of the Fl_Grid widget with additional functionality.

Fl_Grid_Proxy add a list of transient children, i.e. children that are temporarily assigned to a cell that is already taken by another child.

◆ move_cell()

void Fl_Grid_Proxy::move_cell ( Fl_Widget *  in_child,
int  to_row,
int  to_col,
int  how = 0 
)

Move a cell into the grid or within the grid.

If the target cell is already taken, how will determine what to do:

If how is 0, the existing cell at to_row, to_col will be deleted, unlinking the occupant from the grid. in_child will the be inserted at the given location.

If how is 1, the old cell will remain intact, however in_child will be unlinked from the grid.

If how is 2, the old cell will remain intact, and in_child will be removed from the grid, but it will be stored in the transient list and resized to the target cell position and size. If in_child is later moved to an unoccupied cell, it will be removed from the transient list and relinked to the grid. Rowspan and colspan are ignored here.

Parameters
[in]in_childmust already be a child of grid
[in]to_row,to_colmove the child into this cell
[in]how0: replace occupant, 1: don't replace, 2: make transient if occupied
