◆ indent() [2/2]

const char * Fd_Code_Writer::indent ( int  set)

Return a C string that indents code to the given depth.

Indentation can be changed by modifying the multiplicator (*2 to keep the FLTK indent style). Changing spaces to a list of tabs would generate tab indents instead. This function can also be used for fixed depth indents in the header file.

Do not ever make this a user preference, or you will end up writing a fully featured code formatter.

Parameters
[in]setgenerate this indent depth
Returns
pointer to a static string

◆ unique_id()

const char * Fd_Code_Writer::unique_id ( void *  o,
const char *  type,
const char *  name,
const char *  label 
)

Return a unique name for the given object.

This function combines the name and label into an identifier. It then checks if that id was already taken by another object, and if so, appends a hexadecimal value which is incremented until the id is unique in this file.

If a new id was created, it is stored in the id tree.

Parameters
[in]ocreate an ID for this object
[in]typeis the first word of the ID
[in]nameif name is set, it is appended to the ID
[in]labelelse if label is set, it is appended, skipping non-keyword characters
Returns
buffer to a unique identifier, managed by Fd_Code_Writer, so caller must NOT free() it

◆ write_cstring() [2/2]

void Fd_Code_Writer::write_cstring ( const char *  s,
int  length 
)

Write a C string to the code file, escaping non-ASCII characters.

Text is broken into lines of 78 character. FLUID " before and after every line text. A list of control characters and ", ', and \ are escaped by adding a \ in front of them. Escape ?? by writing ?\?. All other characters that are not between 32 and 126 inclusive will be escaped as octal characters.

This function is utf8 agnostic.

Parameters
[in]swrite this string
[in]lengthwrite so many bytes in this string
See also
f.write_cstring(const char*)
